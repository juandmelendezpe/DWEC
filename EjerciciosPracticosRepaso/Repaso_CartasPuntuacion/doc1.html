1. Introducción

El Juego de Cartas es una aplicación web que permite a dos jugadores enfrentarse en un juego de estrategia por turnos utilizando cartas con diferentes estadísticas. Cada jugador crea tres cartas con valores personalizados de fuerza, defensa y habilidad, y luego se enfrentan en batallas donde cada carta tiene fortalezas y debilidades específicas.

Nota: El juego está diseñado para ser jugado en un mismo dispositivo, donde los jugadores se alternan para realizar sus acciones.

2. Reglas del Juego

Principios Básicos:

  • Participan dos jugadores que se enfrentan en un juego de cartas por turnos.
  • Cada jugador crea 3 cartas personalizadas con valores de fuerza, defensa y habilidad.
  • La suma de los valores de cada carta no puede superar los 30 puntos.
  • Los jugadores se turnan para atacar y defender.
  • Durante el combate, las cartas se enfrentan utilizando diferentes estadísticas.
  • La carta con el valor más bajo pierde la batalla y queda fuera del juego.
  • El juego consiste en 3 turnos. Al finalizar, gana el jugador con más cartas restantes.

3. Interfaz de Usuario

3.1 Configuración Inicial

Al iniciar el juego, se muestra una pantalla para ingresar los nombres de los dos jugadores:

  • Campo de texto para el nombre del Jugador 1
  • Campo de texto para el nombre del Jugador 2
  • Botón "Comenzar Juego" para iniciar la partida
Nota: Si no se ingresan nombres, se utilizarán los nombres por defecto "Jugador 1" y "Jugador 2".

3.2 Creación de Cartas

Una vez iniciado el juego, los jugadores crean sus cartas por turnos:

Formulario de Creación de Cartas:

  • Nombre: Campo para ingresar el nombre de la carta
  • Fuerza: Campo numérico para el valor de fuerza
  • Defensa: Campo numérico para el valor de defensa
  • Habilidad: Campo numérico para el valor de habilidad
  • Puntos disponibles: Indicador de puntos restantes (máximo 30)
  • Botón "Crear Carta": Para finalizar la creación de la carta

Importante: Si la suma de los valores supera los 30 puntos, no se podrá crear la carta y se mostrará un mensaje de error.

Después de crear las 3 cartas, el jugador actual tiene la opción de mostrar u ocultar sus cartas mediante un botón "Mostrar Cartas" / "Ocultar Cartas".

3.3 Fase de Combate

Una vez que ambos jugadores han creado sus cartas, comienza la fase de combate:

Elementos de la Interfaz de Combate:

  • Indicador de turno: Muestra de quién es el turno actual
  • Botones de ataque: Opciones para elegir el tipo de ataque (fuerza, defensa, habilidad)
  • Área de cartas: Muestra las cartas disponibles del jugador actual
  • Área de batalla: Muestra las cartas enfrentadas y el resultado
  • Registro de juego: Muestra los eventos importantes del juego
Nota: Las cartas que ya han sido utilizadas y perdieron una batalla aparecerán con una opacidad reducida y no podrán ser seleccionadas nuevamente.

5. Ejemplos de Uso

Ejemplo de Creación de Cartas:

El Jugador 1 (Javier) crea las siguientes cartas:

  • Carta 1: "Guerrero" - Fuerza: 15, Defensa: 10, Habilidad: 5
  • Carta 2: "Mago" - Fuerza: 8, Defensa: 7, Habilidad: 15
  • Carta 3: "Arquero" - Fuerza: 12, Defensa: 8, Habilidad: 10

El Jugador 2 (Jesús) crea las siguientes cartas:

  • Carta 1: "Caballero" - Fuerza: 14, Defensa: 12, Habilidad: 4
  • Carta 2: "Druida" - Fuerza: 9, Defensa: 9, Habilidad: 12
  • Carta 3: "Ninja" - Fuerza: 10, Defensa: 5, Habilidad: 15

Ejemplo de Combate:

Primer Turno:

  • Javier (atacante) elige atacar con "Fuerza"
  • Javier selecciona su carta "Guerrero" con Fuerza = 15
  • Jesús (defensor) selecciona su carta "Caballero" con Defensa = 12
  • Resultado: 15 > 12, la carta "Caballero" de Jesús es derrotada

Segundo Turno:

  • Jesús (atacante) elige atacar con "Habilidad"
  • Jesús selecciona su carta "Ninja" con Habilidad = 15
  • Javier (defensor) selecciona su carta "Mago" con Habilidad = 15
  • Resultado: 15 = 15, empate, ambas cartas permanecen en juego

Tercer Turno:

  • Se elige aleatoriamente, y Javier es seleccionado como atacante
  • Javier elige atacar con "Defensa"
  • Javier selecciona su carta "Arquero" con Defensa = 8
  • Jesús selecciona su carta "Druida" con Fuerza = 9
  • Resultado: 8 < 9, la carta "Arquero" de Javier es derrotada

Resultado Final:

  • Javier tiene 2 cartas restantes (Guerrero y Mago)
  • Jesús tiene 2 cartas restantes (Druida y Ninja)
  • El juego termina en empate

6. Requisitos Técnicos

Para Usuarios:

  • Navegador web moderno (Chrome, Firefox, Safari, Edge)
  • JavaScript habilitado
  • Resolución de pantalla mínima recomendada: 768px de ancho

Para Desarrolladores:

  • Conocimientos básicos de HTML, CSS y JavaScript
  • Editor de código (Visual Studio Code, Sublime Text, etc.)
  • Servidor web local para pruebas (opcional)
Nota: La aplicación está desarrollada como un archivo HTML único con JavaScript y CSS incorporados, por lo que no requiere instalación de dependencias adicionales.

7. Detalles de Implementación

Estructura del Proyecto:

La aplicación está implementada como un único archivo HTML que contiene todo el código necesario para el funcionamiento del juego:

  • HTML: Define la estructura y los elementos de la interfaz de usuario
  • CSS: Proporciona los estilos visuales de la aplicación
  • JavaScript: Implementa la lógica del juego y la interactividad

Componentes Principales:

  • Estado del Juego (gameState): Objeto que almacena toda la información del juego, incluidos los jugadores, cartas, turnos, etc.
  • Elementos DOM (elements): Referencias a los elementos HTML para manipularlos con JavaScript
  • Funciones de Control: Funciones para manejar la lógica del juego (crear cartas, seleccionar ataques, resolver batallas, etc.)
  • Manejadores de Eventos: Funciones que responden a las interacciones del usuario

Características Técnicas:

  • Diseño responsivo que se adapta a diferentes tamaños de pantalla
  • Validación en tiempo real de la suma de puntos de las cartas
  • Sistema de ocultación/visualización de cartas para mantener la privacidad
  • Efectos visuales para indicar el resultado de las batallas
  • Registro de eventos del juego para seguir el progreso

8. Preguntas Frecuentes

¿Cómo se determina el ganador de una batalla?

El ganador se determina comparando el valor de ataque del jugador atacante con el valor de defensa correspondiente del jugador defensor. La carta con el valor más bajo pierde la batalla y queda fuera del juego.

¿Qué ocurre en caso de empate?

Si los valores son iguales, se produce un empate y ambas cartas permanecen en juego.

¿Puedo usar una carta que ya ha sido derrotada?

No, una vez que una carta ha sido derrotada en una batalla, queda marcada como usada y no puede ser seleccionada nuevamente.

¿Por qué no puedo crear una carta con ciertos valores?

La suma de los valores de fuerza, defensa y habilidad no puede superar los 30 puntos. Si intentas asignar valores que exceden este límite, no podrás crear la carta.

¿Cómo se determina quién ataca en el tercer turno?

En el tercer turno, el jugador atacante se elige de forma aleatoria mediante un algoritmo que da a ambos jugadores un 50% de probabilidad de ser seleccionados.

¿Se puede jugar con más de dos jugadores?

No, actualmente el juego está diseñado exclusivamente para dos jugadores.

¿Qué pasa si se refresca la página durante una partida?

Si la página se refresca, se perderá todo el progreso del juego y será necesario comenzar una nueva partida desde el principio.
