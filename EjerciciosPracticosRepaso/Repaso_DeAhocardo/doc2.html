Documentación: Juego del Ahorcado

Aplicación web interactiva construida con HTML, CSS y JavaScript

1. Descripción General

Esta aplicación web implementa el clásico juego del ahorcado, donde los jugadores intentan adivinar una palabra oculta letra por letra. Con cada error, se va dibujando una parte del "ahorcado" hasta completarlo, momento en el cual el jugador pierde si no ha conseguido adivinar la palabra.

La aplicación ha sido desarrollada cumpliendo los requisitos específicos del bloque 3 sobre el uso de Strings, arrays, objetos y manejo de estructuras en JavaScript.

2. Requisitos Implementados

Requisito Descripción Implementación
3.1 Construcción progresiva del muñeco Se utiliza un elemento <pre> con representaciones ASCII del ahorcado que se actualiza según los errores.
3.2 Espacios para letras acertadas Se crean cajas dinámicamente para cada letra de la palabra a adivinar.
3.3 Visualización de letras introducidas Las letras pulsadas se muestran en una fila y se actualiza el contador si se repite una letra.
3.4 Estructura de muñeco en pre-formato Se utiliza un elemento <pre> que respeta los saltos de línea y tabulaciones.
3.5 Lectura de letras desde teclado Se implementa el evento keydown para detectar las pulsaciones del teclado.
3.6 Estructura HTML estática Los elementos mantienen sus posiciones durante el juego.
3.7 Reinicio del juego El juego se reinicia al pulsar F5 o mediante un botón al finalizar la partida.

  • Botón de reinicio: Permite reiniciar el juego al finalizar.

3.2. Estilos CSS

Los estilos CSS proporcionan una interfaz visual atractiva y responsiva:

  • Layout flexible usando Flexbox para adaptarse a diferentes tamaños de pantalla.
  • Estilos específicos para los contenedores, cajas de letras y mensajes.
  • Efectos visuales para mejorar la experiencia de usuario (colores, bordes, etc.).

3.3. Estructura JavaScript

El código JavaScript se organiza de la siguiente manera:

  • Variables globales: Para mantener el estado del juego.
  • Inicialización: Configuración inicial del juego.
  • Gestión de eventos: Captura y procesamiento de pulsaciones de teclas.
  • Funciones de actualización: Actualizan la interfaz según las acciones del usuario.
  • Lógica del juego: Implementa las reglas del juego del ahorcado.

5. Representación del Ahorcado

El ahorcado se representa mediante caracteres ASCII en un elemento <pre>. Los diferentes estados se almacenan en un array y se muestran progresivamente a medida que el jugador comete errores: 

const estadosAhorcado = [
    `  +---+
  |   |
      |
      |
      |
      |
=========`,
    `  +---+
  |   |
  O   |
      |
      |
      |
=========`,
    // ... más estados ...
];

En total hay 7 estados que representan: base inicial, cabeza, cuerpo, brazo izquierdo, brazo derecho, pierna izquierda y pierna derecha.

6. Manejo de Eventos

El juego captura las pulsaciones de teclas mediante el evento keydown: 

document.addEventListener('keydown', manejarTecla);

La función manejarTecla filtra las pulsaciones para aceptar solo letras: 

function manejarTecla(evento) {
    if (juegoTerminado) return;
    
    // Obtener la letra y convertirla a minúscula
    const tecla = evento.key.toLowerCase();
    
    // Verificar si es una letra
    if (/^[a-z]$/.test(tecla)) {
        procesarLetra(tecla);
    }
}

También se implementa un evento para el botón de reinicio: 

botonReinicio.addEventListener('click', function() {
    iniciarJuego();
});

7. Flujo del Juego

El flujo del juego sigue estos pasos:

  1. Inicialización: Se selecciona una palabra aleatoria y se prepara la interfaz.
  2. Entrada del usuario: El jugador pulsa letras en el teclado.
  3. Procesamiento: Se comprueba si la letra está en la palabra y se actualiza el estado del juego.
  4. Actualización de la interfaz: Se actualiza la visualización del ahorcado, la palabra y las letras usadas.
  5. Comprobación de fin de juego: Se verifica si el jugador ha ganado o perdido.
  6. Mensaje final: Se muestra un mensaje indicando el resultado y la opción de reiniciar.

El juego termina cuando:

  • El jugador adivina todas las letras de la palabra (victoria).
  • El jugador comete suficientes errores para completar el ahorcado (derrota).

8. Aspectos Destacados de la Implementación

8.1. Registro de Letras Introducidas

La aplicación mantiene un registro de las letras que el usuario ha introducido y muestra un contador para las letras repetidas: 

// Actualizar contador de letras
if (contadorLetras[letra]) {
    contadorLetras[letra]++;
} else {
    contadorLetras[letra] = 1;
    letrasIntroducidas.push(letra);
}

8.2. Actualización Dinámica de la Interfaz

La interfaz se actualiza dinámicamente mediante manipulación del DOM: 

function actualizarPalabra() {
    contenedorPalabra.innerHTML = '';
    
    for (let i = 0; i < palabraOculta.length; i++) {
        const letraBox = document.createElement('div');
        letraBox.className = 'letter-box';
        letraBox.textContent = palabraOculta[i] !== '_' ? palabraOculta[i] : '';
        contenedorPalabra.appendChild(letraBox);
    }
}

8.3. Selección Aleatoria de Palabras

Las palabras se seleccionan aleatoriamente del array de palabras disponibles: 

palabraActual = palabras[Math.floor(Math.random() * palabras.length)];
